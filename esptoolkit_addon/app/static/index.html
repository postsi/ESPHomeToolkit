ESPToolkit

1. API token

The add-on uses a single API token everywhere (MCP, API, this page). If you have not set one in Settings → Add-ons → ESPToolkit → Configuration, the add-on auto-generates one on first start. To change the token, set api_token in Configuration and restart the add-on.

2. MCP (Cursor / VSCode)

Copy the config below into Cursor. Recommended: use the Direct URL block so Cursor talks to this add-on over HTTP (no extra install). Ensure the URL is reachable from your machine (same network as Home Assistant). Optional: local stdio proxy — pip install git+https://github.com/postsi/ESPHomeToolkit.git (if you use the stdio config).

Regenerate creates a new token and restarts the add-on; then update Cursor mcp.json from the new config below. (If you set api_token in Configuration, that value is used instead.)

Test connection:

No API token set. Set api_token in Add-on Configuration and restart the add-on, then click "Load MCP config" again.
